GIT/GITHUB

Git er et system for distribuert versjonskontroll system. Dette systemet gjør det veldig mye enklere å kunne jobbe sammen med utvikling, og det gjør det enklere å jobbe alene på flere maskiner. Git er veldig essensielt med tanke på at det alltid passer på at man har sikkerhetskopiert innholdet du jobber med. Versjonskontroll vil si at programmet sjekker ting som for eksempel «Når ble fila opprettet?», «Hvorfor har jeg ikke riktig versjon?» og «Hvorfor fungerer det ikke nå?» Dette er til stor hjelp når du skal skrive til en oppgave. Versjonskontrollen kan fungere som Dropbox for videre utvikling. Dette gjør at man kan slå sammen kode som man jobber på til ett prosjekt, og da videre jobbe på forskjellige steder. Et godt argument for å bruke Git er produktets utrolige høye hastighet. Tidligere hvor man kunne bruke et par minutter på å oppdatere og hente filer under versjonskontroll, bruker Git bare noen sekunder på samme operasjon. En annen god grunn til å bruke Git er at en enhver utvikler henter ut sin egen klone av et felles prosjekt. Dette gjør så man kan jobbe med filene selv uten tilgang til en sentral server.

Github er en gratis løsning man kan lagre koden sin på. Dette er kun en tjeneste av Git. Github er et åpent miljø hvor programmerere kan fritt dele og samarbeide på åpen kildekode. Github gjør det enkelt å finne nyttig kode, og sende inn endringer av andre prosjekter. Som et resultat har det blitt hjemmet til nesten enhver åpent kildekode-prosjekt av betydning.
